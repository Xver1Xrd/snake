# TimeEcho Snake — Отчёт о разработке

## 1. Концепция игры

**TimeEcho Snake** — это классическая аркадная игра «Змейка» с инновационной механикой «временных эхо». Игрок управляет змейкой, собирает еду и избегает столкновений со стенами, собственным хвостом и эхо-препятствиями — призрачными копиями прошлых позиций змейки, которые постепенно заполняют игровое поле.

### Основные механики:
- **Три типа еды:** обычная (зелёная, +10 очков), кристалл времени (синий, замедляет время на 5 секунд) и призрачная сфера (фиолетовая, позволяет проходить сквозь эхо 5 секунд).
- **Эхо-препятствия:** на каждой 3-й клетке перемещения на случайной позиции тела змейки оставляется призрачная копия. Они постепенно накапливаются, усложняя игру.
- **Прогрессия сложности:** каждые 8 секунд скорость игры увеличивается (интервал движения уменьшается на 8%, до минимальных 0.06 сек).
- **Дополнительные эффекты:** тряска экрана при смерти, частицы при съедании еды, пульсирующая подсветка эхо.

### Управление:
- WASD / стрелки — движение
- ESC — пауза / меню
- Space / Enter — выбор / рестарт

## 2. Архитектура и технологии

### Стек:
- **Язык:** C++17
- **Графика/ввод:** raylib 5.0
- **Сборка:** CMake 3.16+
- **Веб-сборка:** Emscripten 5.0.7 (WebAssembly)
- **Платформы:** Linux (нативная), Web (WASM)

### Структура классов:

```
src/
├── main.cpp      — точка входа, инициализация окна, главный цикл
├── Game.cpp/h    — игровая логика, меню, система эхо, частицы
├── Snake.cpp/h   — класс змейки (движение, коллизии, отрисовка)
```

**Snake** (`Snake.h`):
- Хранит тело как `std::deque<Vector2>`
- Реализует движение, рост, самопересечение
- Очередное направление (`nextDir`) для предотвращения реверсивного движения

**Game** (`Game.h`):
- Состояния: MENU, PLAYING, GAME_OVER, CONTROLS
- Еда: `std::vector<FoodItem>` с типом (NORMAL, TIME_CRYSTAL, GHOST_ORB)
- Эхо: `std::vector<EchoObstacle>` с альфа-каналом и пульсацией
- Частицы: `std::vector<Particle>` с гравитацией и затуханием
- Сохранение рекорда через файл `timeecho_highscore.dat`

### Сборка:
- Нативный бинарник: `cmake -S . -B build && cmake --build build`
- WebAssembly: `cmake -S . -B build-web -DCMAKE_TOOLCHAIN_FILE=.../Emscripten.cmake && cmake --build build-web`
- При сборке для WASM raylib подключается через FetchContent (клонируется и компилируется под Emscripten)

## 3. Процесс разработки

Разработка велась итеративно, каждый этап добавлял новую функциональность:

### Этап 1: Базовая змейка
Реализован класс Snake с движением по сетке, сменой направления, ростом при съедании еды. Отрисовка через raylib. Простейший игровой цикл с таймером движения.

### Этап 2: Система меню и состояний
Добавлены состояния Game (MENU, PLAYING, GAME_OVER). Реализовано главное меню с тремя опциями: Play, Controls, Quit. Экран Game Over с отображением счёта и рекорда.

### Этап 3: Еда трёх типов
Введены три типа еды со случайным распределением (60% обычная, 20% временной кристалл, 20% призрачная сфера). Каждый тип имеет уникальный визуальный стиль (цвет, пульсация, форма). Реализованы эффекты: замедление времени и прохождение сквозь эхо.

### Этап 4: Система эхо-препятствий
Самая сложная часть. Эхо появляются на прошлых позициях змейки каждые ~3 хода. Они не должны блокировать еду, накладываться друг на друга или появляться на голове змейки. Реализовано плавное затухание (alpha -= 0.001) и пульсация (sin-волна). При достижении лимита в 40 эхо — старые удаляются.

### Этап 5: Частицы и визуальные эффекты
Добавлена система частиц: взрыв при съедании еды (8-12 частиц с гравитацией, случайным направлением и затуханием). Тряска экрана при смерти. Подсветка выбранного пункта меню с пульсацией. Анимированные звёзды на фоне меню.

### Этап 6: Прогрессия сложности
Автоматическое увеличение скорости каждые 8 секунд. Визуальный индикатор скорости на HUD. Сохранение лучшего счёта в файл.

### Этап 7: WebAssembly сборка
Адаптация кода для компиляции под Emscripten. Добавлен `#include <emscripten.h>` в main.cpp. Главный цикл через `emscripten_set_main_loop`. Использование `--shell-file` для кастомного HTML.

## 4. Проблемы и их решения

### Проблема 1: Эхо-препятствия блокируют еду
**Ситуация:** Эхо появлялись на клетках, где уже была еда, делая её недоступной.
**Решение:** При спавне эхо проверяется, не занята ли клетка едой, телом змейки (кроме головы) или другим эхо. При совпадении — эхо не создаётся.

### Проблема 2: Эхо накладываются друг на друга
**Ситуация:** Множественные эхо на одной клетке создавали визуальный шум.
**Решение:** Добавлена проверка при спавне — если клетка уже содержит эхо, новое не ставится.

### Проблема 3: Замедление времени влияет на отрисовку меню
**Ситуация:** При включённом замедлении времени анимации меню тоже замедлялись.
**Решение:** Эффект времени применяется только в состоянии PLAYING. В Update() сначала проверяется состояние, затем применяется timeScale.

### Проблема 4: Долгая сборка WebAssembly (Emscripten)
**Ситуация:** FetchContent клонирует и компилирует raylib с нуля при каждой cmake-настройке. Emscripten также компилирует системные библиотеки (libc, libc++, etc.) при первом запуске, что занимает 5+ минут.
**Решение:** Для ускорения raylib собран отдельно (`/tmp/raylib-wasm-build`), установлен (`/tmp/raylib-wasm-install`), и проект компилируется напрямую через `em++` без CMake, подключая уже готовую статическую библиотеку. Повторные сборки значительно быстрее.

### Проблема 5: Canvas undefined в браузере
**Ситуация:** При загрузке страницы GLFW пытался найти canvas в DOM, но его не было, что вызывало ошибку `TypeError: canvas is undefined` в _glfwInit.
**Решение:** В shell.html добавлен элемент `<canvas id="canvas">` явно. Emscripten находит его через `document.getElementById('canvas')` и использует для рендеринга.

### Проблема 6: Бесконечный экран загрузки
**Ситуация:** Текст "Loading TimeEcho Snake..." оставался навсегда, даже после загрузки WASM.
**Решение:** В shell.html добавлен JavaScript-интервал, который отслеживает появление canvas с ненулевыми размерами (после инициализации raylib) и скрывает/удаляет loading-элемент.

### Проблема 7: Сохранение рекорда в WASM
**Ситуация:** Игра пишет рекорд в файловую систему через fopen/fprintf. В браузере это работает через виртуальную FS Emscripten, но данные не сохраняются между сессиями.
**Решение:** В текущей версии рекорд хранится в памяти и сбрасывается при обновлении страницы. Для постоянного хранения требуется интеграция с localStorage (через Emscripten IDBFS или JavaScript-прокси). Оставлено как потенциальное улучшение.

## 5. Веб-сборка и деплой

### Процесс сборки WASM:

```bash
source emsdk/emsdk_env.sh
# Сборка raylib под WASM (однократно)
cmake -S /tmp/raylib-wasm -B /tmp/raylib-wasm-build \
  -DCMAKE_TOOLCHAIN_FILE=$EMSDK/upstream/emscripten/.../Emscripten.cmake \
  -DPLATFORM=Web
cmake --build /tmp/raylib-wasm-build
cmake --install /tmp/raylib-wasm-build --prefix /tmp/raylib-wasm-install

# Сборка игры
em++ -std=c++17 src/*.cpp /tmp/raylib-wasm-install/lib/libraylib.a \
  -I/tmp/raylib-wasm-install/include -I src \
  -s USE_GLFW=3 -s ASYNCIFY -s TOTAL_MEMORY=64MB \
  --shell-file web/shell.html -o build-web/index.html -O3
```

### Результат:
- `index.html` (852 B) — страница с canvas и loading-экраном
- `index.js` (173 KB) — JavaScript-прокладка Emscripten (GLFW, Browser, FS и т.д.)
- `index.wasm` (184 KB) — скомпилированный WebAssembly-модуль

## 6. Заключение

TimeEcho Snake — полноценная аркадная игра с оригинальной механикой эхо-препятствий, тремя типами еды с уникальными эффектами и прогрессией сложности. Проект успешно скомпилирован как нативный бинарник (Linux, raylib) и как WebAssembly-модуль (Emscripten). В ходе разработки решены ключевые проблемы: корректное размещение эхо без блокировки еды, независимость временных эффектов от интерфейса, инициализация canvas в браузере и скрытие экрана загрузки.

### Потенциальные улучшения:
- Постоянное сохранение рекорда в браузере (localStorage через IDBFS)
- Мобильное управление (touch/swipe)
- Звуковые эффекты
- Таблица лидеров
- Разные режимы игры (классический, бесконечный, головоломка)
